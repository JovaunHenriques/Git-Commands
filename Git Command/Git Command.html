Git Commmands


Have you every got stumped by a git command or just dont know how to use them here is a few..

    • Example: git add file.txt (Stage changes in the file file.txt for the next commit.)
  • ðŸ’ git branch
    • Lists, creates, or deletes branches. It helps manage different lines of development in your project.
    • Example:
      • git branch (List all branches.)
      • git branch new-feature (Create a new branch named new-feature.)
      • git branch -d old-feature (Delete the branch old-feature.)
  • ðŸ’ git checkout
    • Switches between branches, commits, or tags, allowing you to work on a specific snapshot of your project.
    • Example:
      • git checkout main (Switch to the main branch.)
      • git checkout some-commit-hash (Switch to a specific commit using its hash.)
  • ðŸ’ git pull
    • Fetches changes from a remote repository and integrates them into your local branch.
    • Example: git pull origin main (Fetch and merge changes from the main branch on the remote repository.)
  • ðŸ’ git push
    • Sends your committed changes to a remote repository, updating it with your latest work.
    • Example: git push origin new-feature (Push the local commits from the new-feature branch to the remote repository.)
  • ðŸ’ git rebase
    • Integrates changes from one branch onto another by moving, applying, or combining commits.
    • Example: git rebase main (Reapply commits from the current branch on top of the main branch.)
  • ðŸ’ git rebase -i
    • Interactively rewrites commit history, allowing you to squash, edit, reorder, or drop commits.
    • Example: git rebase -i HEAD~3 (Interactively rebase the last 3 commits to modify, squash, or reorder them.)
  • ðŸ’ git reflog
    • Displays a log of all references (branches, commits, etc.) that have been updated, helping you recover lost work.
    • Example: git reflog (View a history of recent branch updates and commits, useful for undoing mistakes.)
  • ðŸ’ git status
    • Shows the current state of your working directory, indicating which files are modified, staged, or untracked.
    • Example: git status (Display the status of tracked and untracked files in your repository.)
  • ðŸ’ git remote
    • Manages remote repositories, allowing you to add, remove, or list remotes associated with your project.
    • Example:
      • git remote -v (List the URLs of all remote repositories.)
      • git remote add upstream <upstream_repo_url> (Add a remote named upstream with the specified URL.)
  • ðŸ’ git merge
    • Combines changes from different branches, integrating them into the current branch.
    • Example: git merge feature-branch (Merge changes from the feature-branch into the current branch.)
