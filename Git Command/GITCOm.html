Git Commmands

Have you every got stumped by a git command or just dont know how to use them here is a few..

  • git add
      • Adds changes from your working directory to the staging area, preparing them for the next commit.

    • git branch
        • Description: Lists, creates, or deletes branches. It helps manage different lines of development in your project.

      • git checkout
          • Description: Switches between branches, commits, or tags, allowing you to work on a specific snapshot of your project.

        • git pull
            • Description: Fetches changes from a remote repository and integrates them into your local branch.

          • git push
              • Description: Sends your committed changes to a remote repository, updating it with your latest work.

            • git rebase
                • Description: Integrates changes from one branch onto another by moving, applying, or combining commits.

              • git rebase -i
                  • Description: Interactively rewrites commit history, allowing you to squash, edit, reorder, or drop commits.

                • git reflog
                    • Description: Displays a log of all references (branches, commits, etc.) that have been updated, helping you recover lost work.

                  • git status
                      • Description: Shows the current state of your working directory, indicating which files are modified, staged, or untracked.

                    • git remote
                        • Description: Manages remote repositories, allowing you to add, remove, or list remotes associated with your project.

                      • git merge
                          • Description: Combines changes from different branches, integrating them into the current branch.
